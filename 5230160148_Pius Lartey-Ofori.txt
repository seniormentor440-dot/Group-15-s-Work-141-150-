
1)i) Git, GitHub, Version Control Systems, and Code Push Steps
Git is a distributed version control system used to track changes in source code during software development. It helps developers manage different versions of a project, collaborate with others, and restore previous versions when mistakes occur. Git works locally on a developer’s computer and stores a complete history of all changes made to files, which makes it fast, reliable, and efficient.


ii) GitHub is a web-based platform that hosts Git repositories online. It allows developers to store their code remotely and work together through features such as pull requests, issue tracking, and documentation. GitHub is commonly used for team projects and open-source software development.


iii)A Version Control System (VCS) is a tool that records file changes over time. It enables multiple users to work on the same project without conflicts, reduces the risk of data loss, and improves accountability by tracking who made changes and when. Git is an example of a distributed version control system.
The main difference between Git and GitHub is that Git is a local version control tool, while GitHub provides online hosting and collaboration services for Git repositories.



2)Steps to Push Code to GitHub
Install Git, initialize a repository using git init, add files with git add ., commit changes using git commit -m "message", create a repository on GitHub, link the local repository, and push the code using git push.
