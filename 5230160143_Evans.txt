What is Git?
Git is a distributed version control system created in 2005 by Linus Torvalds. It allows developers to track code changes, create branches to work on new features, and merge those features back into the main project. Because Git is distributed, every user has a complete copy of the project history on their computer. This makes it fast, reliable, and usable even without internet access.

What is Github?
GitHub is a web-based platform that hosts Git repositories online. It provides collaboration tools such as pull requests, issue tracking, and project management features. GitHub makes it easier for teams to work together and share projects publicly or privately.

What is Version Control System (VCS)?
A Version Control System (VCS) is a tool that helps developers track and manage changes made to files over time. It records every modification, allowing users to review history, compare versions, and restore earlier work if needed. Version control is especially important in software development because multiple people can work on the same project without overwriting each other’s changes. There are centralized systems and distributed systems.

Difference between Git and GitHub
The main difference between Git and GitHub is that Git is the software used to track and manage changes locally, while GitHub is an online service that stores Git repositories and supports collaboration. Git works offline; GitHub requires internet access.
