
(A) What Git is about
Git is a distributed version control system used to track changes in source code during software development. It was created in 2005 by Linus Torvalds to help manage the development of the Linux. Git allows developers to record changes to files, revert to previous versions, and work on different features using branches. Because it is distributed, every developer has a full copy of the project history on their local machine. This makes Git fast, reliable, and suitable for collaborative development.

(B) What GitHub is
GitHub is a web-based platform that hosts Git repositories online. It provides tools for collaboration such as pull requests, issue tracking, code reviews, and project management. GitHub allows developers to store their code in the cloud and work together from different locations. It also supports open-source projects and private repositories for teams and organizations.

(C) What a Version Control System is
A Version Control System (VCS) is a system that records changes to files over time so that specific versions can be recalled later. It helps developers track modifications, compare changes, restore previous versions, and collaborate without overwriting each other’s work.

(D) Difference Between Git and GitHub
Git is a tool used to manage and track code changes locally on a computer. GitHub is an online service that hosts Git repositories and provides collaboration features. In simple terms, Git is the software, while GitHub is the platform that uses Git.

2. Steps Involved in Pushing Code from Local Machine to GitHub

1.Install Git on your computer.
2.Create a repository on GitHub.
3.Open your project folder in the terminal.
4.Initialize Git:
"git init"
5.Add files to staging area:
"git add"
6.Commit the changes:
git commit -m "Initial commit"
7.Connect to GitHub repository:
git remote add origin https://github.com/username/repository.git
8.Push the code to GitHub:
git push -u origin main
